# TSR Studio 2.0

**中药多组学、活性成分与作用靶标深度分析平台**

开发者：**河北中医药大学药学院 张丹**
支持系统：Windows x64、macOS Intel、macOS Apple Silicon（M1/M2/M3/M4/M5）

> TSR Studio 2.0 面向中药药效、化学成分、转录组、蛋白组、代谢组、菌群、短链脂肪酸以及行为学/生化指标的整合分析。软件把研究设计、数据质控、差异分析、深度挖掘、候选成分—靶标筛选、分子对接、分子动力学结果整理和论文数据导出放在同一个可追溯项目中。

## 一、主要功能

### 1. 研究设计与数据管理

- 设置项目名称、疾病或动物模型、物种、空白组、模型组和一个或多个给药组。
- 建立 `Sample ID—Animal ID—Group—Batch—Tissue—Timepoint` 对应关系。
- 自动识别不同文件中的样本列并检查跨组学样本是否匹配。
- 支持同一项目保存、关闭和继续分析；Windows 与 macOS 共用项目格式。
- 记录数据文件、分析参数、软件引擎、运行时间、结果状态和限制信息。

### 2. 可导入的数据类型

| 模块 | 推荐输入数据 | 主要用途 |
|---|---|---|
| 实测化学成分 | LC–MS定量矩阵、成分名称、m/z、RT、样本峰强度 | 活性成分筛选、成分丰度比较 |
| 转录组 | raw counts矩阵 | DESeq2差异分析、GO/KEGG/GSEA、WGCNA |
| 蛋白组 | DIA/DDA/TMT定量矩阵 | limma差异分析、富集、WGCNA、靶标筛选 |
| 代谢组 | 峰表、代谢物注释或靶向定量矩阵 | 差异代谢物及跨组学关联 |
| 菌群 | ASV/OTU/物种丰度矩阵 | 菌群差异及表型关联 |
| 短链脂肪酸 | 个体浓度或定量结果 | 表型及多组学关联 |
| 药效与生化 | 行为学、生化、病理等个体数据 | 组间比较、模块性状及成分—活性关联 |

矩阵数据通常采用“一行一个特征、一列一个样本”。样本列名必须与样本关系表的 `Sample_ID` 一致。表型文件第一列建议命名为 `Sample_ID`，其余列为数值型指标。

### 3. 基础统计分析

- 数据结构与缺失率检查。
- 描述统计、PCA、组间差异比较、效应量与 BH-FDR。
- 转录组调用 R/DESeq2；仅接受非负整数 raw counts，TPM/FPKM 不会冒充 DESeq2 输入。
- 蛋白组调用 R/limma 经验贝叶斯模型。
- 模型组相对空白组、给药组相对模型组的方向逆转和恢复率分析。
- 所有给药组两两直接比较，例如炮制品相对生品、处方B相对处方A或药物B相对药物A；结果单独标记为“给药组间比较”。
- 严格显著结果与探索性候选分层保存，不用普通 t 检验伪装缺失的正式运行环境。

### 4. 深度生物信息学与机器学习

- **GO富集**：BP、CC、MF，BH-FDR及ID映射率。
- **KEGG富集**：按物种进行通路富集，并保留完整背景和输入基因。
- **GSEA**：使用全量有方向的排序结果，不依赖任意差异阈值。
- **WGCNA**：软阈值选择、共表达模块、模块成员度、模块—性状关联及Hub候选。
- **分子—表型关联**：用户可选择一个或多个行为学、生化或药效指标，计算 Spearman 相关与 BH-FDR。
- **机器学习共识筛选**：Elastic Net、随机森林、SVM、分层交叉验证、ROC AUC、平衡准确率、置换检验和特征重要性。
- **集合交叠**：输出 Venn、花瓣和 UpSet 所需的严格显著集合、精确交集和全部成员。
- **跨组学相关网络**：在共同样本上计算跨数据块相关及FDR，保留可追溯网络边。

GO、KEGG和GSEA可在“深度挖掘”页面指定比较，例如选择 `CG_vs_G`，只对CG相对G的差异结果进行富集，不与其他比较混合。

WGCNA 少于15个独立样本时直接阻断；15–29个样本标记为探索性。机器学习至少需要两组且每组不少于3个独立样本；总样本较小时仅用于候选排序，不能替代独立验证。

### 5. 活性成分与候选靶标筛选

软件不会把所有成分与所有蛋白机械地全排列。候选证据来自：

1. 成分或代谢物的实测丰度及组间变化；
2. 成分丰度与药效/行为学指标的相关性；
3. 转录组、蛋白组中“模型异常—给药逆转”的方向证据；
4. 跨组学一致性、FDR、效应量和恢复率；
5. Open Targets 疾病关联证据及查询时间；
6. PubChem 成分身份、CID、SMILES 和 InChIKey；
7. 后续分子对接或动力学证据。

用户可以在“深度挖掘”页面勾选参与筛选的组学和具体药效指标。只想使用其中几个行为学或生化指标时，仅勾选相应指标即可；未勾选时默认使用全部可识别的数值型表型。

### 6. 分子对接与分子动力学

- 调用本机 AutoDock Vina，保存全部构象评分和 PDBQT 结果。
- 调用本机 GROMACS，整理 RMSD、RMSF、回转半径、SASA 和氢键时间序列。
- 支持生产模拟的检查点续算。

Vina 与 GROMACS 是可选的外部程序。未安装时不影响基础统计、富集、WGCNA和机器学习分析。

## 二、输出内容

### 完整 Excel

包括：项目设计、样本关系、原始导入、QC、方法参数、描述统计、全部比较、严格显著结果、探索性候选、方向逆转、候选成分、候选靶标、成分—靶标证据对、对接/动力学结果及所有深度分析表。

### 论文图包

- SVG可编辑矢量图；
- 高分辨率PNG；
- 每张图对应的TSV源数据；
- 中英文可用的图注文件；
- PCA、火山图、热图、逆转图、富集气泡图、WGCNA模块—性状热图、机器学习比较图、相关图和花瓣图等。

图形采用白底、深蓝—金色结构和色盲友好配色，面向论文排版，但“软件生成”不等于不经人工审核即可投稿。

### 其他输出

- 多组学候选调控网络；
- 深度分析专用Excel与论文图包；
- Markdown自动研究报告；
- 可继续打开的TSR项目文件。

## 三、运行环境

### Python

需要 Python 3.11 或更高版本，并安装：

```text
pandas
scipy
scikit-learn
```

### R/Bioconductor

建议 R 4.3 或更高版本。基础与深度分析需要：

```text
DESeq2
limma
edgeR
jsonlite
WGCNA
clusterProfiler
org.Mm.eg.db
org.Rn.eg.db
org.Hs.eg.db
```

软件中的“检测运行环境”会显示实际找到的 Python、R 路径及依赖状态。macOS 安装包提供 R 依赖安装脚本；Windows 源码中提供 `scripts/deploy-r-windows.ps1`。

## 四、从 GitHub 构建安装包

1. 将本项目源码上传到 GitHub 仓库根目录，确保可以看到 `package.json`、`src`、`electron`、`analysis-runtime` 和 `.github`。
2. 打开仓库的 **Actions** 页面。
3. 选择 **Build Windows and macOS installers**。
4. 点击 **Run workflow**。
5. 构建完成后，在该次运行页面底部下载 Artifacts：
   - `TSR-Studio-2.1.1-Windows-x64-unsigned`
   - `TSR-Studio-2.1.1-macOS-Intel-x64-unsigned`
   - `TSR-Studio-2.1.1-macOS-Apple-Silicon-arm64-unsigned`

也可以创建 `v2.1.1` 标签，工作流会同时建立 GitHub Release。

## 五、安装与首次启动

### macOS Apple Silicon

M1/M2/M3/M4/M5 选择 `macOS-Apple-Silicon-arm64`。下载并解压 Artifact，优先使用 DMG，将应用拖入“应用程序”。

当前构建未使用 Apple 开发者证书签名和公证。首次打开若被系统拦截：

1. 将应用复制到“应用程序”，不要直接在DMG中运行；
2. 在访达中按住 Control 点击应用，选择“打开”；
3. 或进入“系统设置—隐私与安全性”，确认打开该应用。

如果系统明确提示“应用已损坏”，先核对下载是否完整。未签名研究版可在终端执行：

```bash
xattr -dr com.apple.quarantine "/Applications/TSR Studio.app"
```

不要对来源不明的软件执行该命令。

### Windows

下载 Windows Artifact 后，可选择 NSIS 安装程序或 portable 便携版。未签名版本可能触发 SmartScreen，需要确认文件来自自己的 GitHub Actions 构建记录。

## 六、推荐分析流程

1. 新建项目并填写疾病、物种和实验组别；
2. 导入或建立样本关系表，核对每只动物的独立编号；
3. 依次导入表型、化学成分和各组学矩阵；
4. 查看质控提示，处理样本名、缺失率和数据类型问题；
5. 运行各单组学基础分析；
6. 在“深度挖掘”中选择组学及药效指标，运行富集、WGCNA、相关或机器学习；
7. 在“成分—靶标验证”中核对候选证据，必要时补充疾病数据库证据；
8. 根据研究需要运行对接和动力学；
9. 导出完整Excel、论文图包、调控网络和自动报告；
10. 对关键候选进行靶向定量、独立样本或干预实验验证。

## 七、科学解释与使用边界

- 相关性不等于因果关系。
- 数据库预测不等于标准品鉴定或直接结合证据。
- 小样本WGCNA和机器学习只能用于探索性候选排序。
- 富集结果受物种、ID类型、背景基因和数据库版本影响。
- 软件不会自动解决MNAR缺失、批次混杂、伪重复或错误实验设计。
- 对接分数不能单独证明真实结合；动力学结果依赖受体结构、配体参数、力场、体系构建和平衡方案。
- 所有图表和结论在用于论文前均应由研究者复核。
