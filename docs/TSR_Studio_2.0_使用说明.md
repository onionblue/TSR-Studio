# TSR Studio 2.0 使用说明

软件名称：TSR Studio 2.0（Traditional Chinese Medicine Systems Research Studio，中药系统研究工作室）  
署名：河北中医药大学药学院 张丹

## 可完成的任务

TSR Studio 2.0 用于中药药效、化学成分和多组学数据的一体化管理与候选机制挖掘。基础分析包括样本质控、描述统计、PCA、差异分析、火山图、热图和模型—给药方向逆转；深度分析包括 GO、KEGG、GSEA、WGCNA、分子—行为/生化指标相关、机器学习共识筛选、集合交叠及跨组学相关网络。所有分析保留参数、运行引擎、全量表和限制提示。

## 需要导入的数据

每个数据矩阵一行一个特征、一列一个样本。样本列名必须能与样本设计表中的 `Sample_ID` 一一对应。

- 样本设计：`Sample_ID`、`Group`、可选 `Animal_ID`、批次、性别等。
- 转录组：第一列基因 ID，后续为 raw counts；DESeq2 不接受 TPM/FPKM 代替原始计数。
- 蛋白组、代谢组、化学成分、菌群：第一列特征 ID，后续为各样本定量值。
- 行为学/生化指标：第一列 `Sample_ID`，后续每列一个数值型指标。
- 成分—靶标：需要可追溯的成分 ID、靶标 ID 和证据来源；软件不会把数据库预测自动表述为实验验证。

## 深度分析如何选数据

进入“深度挖掘”，先勾选参与分析的组学，再勾选希望作为性状的行为/生化指标。未勾选指标时使用全部数值型表型。

- GO/KEGG：使用已完成的差异分析结果，默认以 FDR < 0.05 的特征进入 ORA，并使用全部检测到的特征作为背景。
- GSEA：使用全量有方向的排序列表，不依赖任意显著性阈值。
- WGCNA：选择一个表达矩阵和性状。少于 15 个独立样本直接阻断；15–29 个样本标记为探索性；建议总样本数至少 30。不得把同一样本的技术重复当作独立样本。
- 机器学习：至少两组、每组至少 3 个独立样本；执行 Elastic Net、随机森林和 SVM 的分层交叉验证、置换检验与特征重要性。样本少于 50 时只用于候选排序，不作为独立验证。
- 分子—表型关联：只计算勾选指标，输出 Spearman r、P 值与 BH-FDR。
- 跨组学网络：只使用共同样本，输出相关系数、P 值和 FDR；相关关系不等同于因果关系。
- Venn/花瓣/UpSet 源数据：使用各组学/比较中通过严格 FDR 阈值的集合，完整成员表均可导出。

## 输出内容

- “完整 Excel”包含原始导入、QC、全部比较、严格显著、探索性结果、逆转结果、候选成分/靶标、对接/动力学和全部深度分析表。
- “论文图包”包含 SVG、PNG、每图源数据和图注；SVG 用于排版，PNG 用于预览或常规投稿。
- “深度分析 Excel”按方法分别保存参数、摘要、提示和全量结果表。
- 项目文件保存全部输入、结果和分析记录，可再次打开继续分析。

## 运行环境

Python 需要 pandas、scipy、scikit-learn。R 基础分析需要 DESeq2、limma、edgeR；深度分析另需 jsonlite、WGCNA、clusterProfiler，以及对应物种注释包。macOS 安装包内提供 R 依赖安装脚本；Windows 可运行 `scripts/deploy-r-windows.ps1`。

## 科学解释边界

软件输出的是可追溯的候选证据，不自动证明活性成分、直接靶点或因果机制。小样本机器学习、WGCNA、数据库富集和相关网络必须结合独立样本、靶向定量、干预实验或结合实验验证。
